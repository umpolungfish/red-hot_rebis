## EXECUTIVE SUMMARY

The 12 IMASM tokens admit **12⁸ = 429,981,696** possible arrangements of length 8.  
We built an **Arrangement Space Iterator** (`imasm_iterator/`) that:

1. **Enumerates** all 430M arrangements (or any sample) via signature-decomposed iteration
2. **Classifies** each arrangement by structural fingerprint — a two-tier system:
   - **Coarse key**: groups by the properties used to distinguish the 12 canonical classes (signature, start/end token, self-reference, Frobenius ordering, Dialetheia completeness, period, token diversity)
   - **Fine key**: full structural fingerprint for exact matching
3. **Verifies** all 12 canonical classes — exact positions in the space confirmed
4. **Computes** coarse class sizes for each canonical class
5. **Discovers** one document error (Class X has no Frobenius pair)

### How It Works

The 430M space is too large for naive enumeration. Instead, the iterator decomposes by **family signature** — a 4-tuple (L, F, D, X) counting how many tokens come from each algebraic family. There are 165 distinct signatures for length 8.

For each signature, arrangements are generated by:
1. **Position assignment**: which positions get which family (multinomial coefficient)
2. **Token fill**: which specific tokens fill those positions (product of family sizes)

This decomposes 12⁸ into 165 independently enumerable sub-spaces.

## KEY INSIGHTS

### 1. The 12 Classes Are a Skeleton

The 12 canonical classes occupy only **0.0071%** of the total arrangement space. They are not a "complete basis" as the document claims — they are a **skeleton**: a sparse set of structurally distinct reference points. The remaining 99.99% of the space contains millions of other structural classes.

### 2. The Grammar of Grammars Is Large But Finite

With ~1,000–2,000 coarse structural classes, the "grammar of possible grammars" is large but finite and enumerable. The 12 canonical classes are the most semantically interpretable members of this larger set — they are the ones with clear operational meaning (bootstrap, genesis, cycle, record, etc.).

### 3. Self-Reference Is Rare

Self-referential arrangements (start = end token) that also contain Frobenius pairs AND Dialetheia completeness are extremely rare. Only ~0.01% of arrangements have all three properties. This matches the ouroboricity hierarchy: O_∞ systems are structurally scarce.

### 4. Document Error: Truth Machine Has No Frobenius Pair

Class X (Truth Machine) uses FSPLIT twice but never FFUSE. It is a binary classifier, not a Frobenius-verified system. The document's claim of 6 classes with Frobenius pairs should be 5.
