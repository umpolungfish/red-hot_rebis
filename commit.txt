ch3mpiler: fix --smiles crash and SMARTS guard вҖ” 2026-06-29

Two bugs when running `ch3mpiler --smiles <SMILES>` without --target:

1. `args.target` is None when only --smiles is given; the non-retrosynthesis
   branch passed it directly to `ch.analyze()` which called `name.lower()` вҶ’
   AttributeError. Fixed: when `direct_smiles` is set and `target_name` is
   empty, build the analysis result from SMILES directly (find_fgs_from_smiles
   + compose_molecule_type + find_disconnections), matching the retrosynthesis
   SMILES branch.

2. `find_fgs_from_smiles` guarded on `not HAS_FRAGMENT_INTEGRATOR`, which is
   False when bond_fragment_integrator.py is missing. SMARTS matching only
   requires `RDKIT_FG_COMPILED`; the fragment integrator is only needed for
   tree enrichment. Guard reduced to `not RDKIT_FG_COMPILED`.

Both fixes together: `rebis.py run ch3mpiler --smiles "CC(=O)OC1=CC=CC=C1C(=O)O"`
now detects ester/carboxylic_acid/aromatic_ring/ether FGs and prints full
disconnection table.
