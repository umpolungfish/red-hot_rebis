Cleanup phase I: unify the facade version with the package version

rebis/__init__.py hard-coded __version__ = "3.0.0" while the package built as
4.0.0, so the two drifted. It now reads importlib.metadata.version(
"red-hot-rebis") with a 4.0.0 fallback, so the facade version tracks the
installed package automatically and cannot drift again. rebis.py carries no
version string (the earlier v2.1 note was stale). Left rhr_p4rky at its own
0.2.0: it is the kernel component and legitimately versions independently.
